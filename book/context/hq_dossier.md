# Штабное досье проекта «Под Огромной Луной»

Этот файл нужен для быстрого входа в проект в новом чате или новой рабочей ветке.
Его задача — не пересказывать роман целиком, а удерживать рабочий контекст, структуру проекта и текущие процессы.

## Что это за проект

«Под Огромной Луной» — роман и одновременно растущая вселенная с несколькими рабочими контурами:
- текст романа
- редакторская работа по главам
- карта сезона
- сайт романа
- визуальная система
- ролики к главам
- в перспективе — короткие пересказы и видео для соцсетей

## Главные принципы работы

- Не смешивать разные типы задач в одном чате.
- Один чат = одна рабочая нить.
- Для новых глав лучше отдельный чат на каждую главу.
- Для больших тем — отдельные постоянные чаты:
  - структура сезона
  - сайт
  - визуальная система
  - ролики к главам
- Опора не на память чата, а на файлы проекта.
- Не плодить красивую теорию без выхода в рабочие файлы.
- Не льстить и не хвалить из вежливости.
- Сначала строить каркас, потом расширять.

## Как открывать новые рабочие чаты

### 1. Новая глава
Использовать отдельный чат на одну главу.

Старт:
- прочитать `AGENTS.md`
- прочитать `chapter_index.md`
- прочитать контекстные файлы
- работать только с одной главой

Конвейер:
1. snapshot
2. titles + scene tags + knowledge tags
3. editorial
4. при необходимости round2
5. update index
6. rebuild chapters_full.json

### 2. Структура сезона
Отдельный постоянный чат.

Там делать:
- где кончается начало
- где главный бум
- линии персонажей
- кто готов к промежуточному финалу

### 3. Сайт
Отдельный постоянный чат.

Там делать:
- структура данных
- страницы
- фильтр
- подключение `chapters_full.json`
- аккуратная полировка интерфейса

Не смешивать с:
- визуальной библией
- сезонной редактурой
- новыми главами

### 4. Визуальная система
Отдельный чат.

Там делать:
- визуальные зоны мира
- цвет
- свет
- фактуры
- логика сайта как образа мира
- персонажные пакеты

### 5. Ролики к главам
Отдельный чат.

Там делать:
- видеоязык
- карточки роликов
- production brief
- prompts
- refs
- шоты

Не обсуждать там сайт и не делать сезонную редактуру.

## Что важно помнить про сайт

Первая читательская версия сайта сейчас опирается только на:
- указатель глав
- страницу главы
- фильтр по сценическим тегам
- тип главы при необходимости

На сайт не выводятся:
- теги базы
- функция главы
- внутренние редакторские штуки
- статусы как часть читательского интерфейса

## Что важно помнить про ролики

Карточка ролика — не финальный prompt, а промежуточный рабочий документ.

Нормальный конвейер ролика:
1. рабочая карточка главы
2. короткий production brief
3. English prompt
4. negative prompt
5. список refs
6. shot prompts
7. генерация
8. запись выводов в `video_production_notes.md`

## Что важно помнить про референсы

Референсы должны храниться системно, а не хаотично.

Принцип именования:
- `base_01.jpg`
- `closeup_01.jpg`
- `ritual_01.jpg`

Смысл:
- `base` — канонический вид
- `closeup` — лицо и эмоция
- `ritual` — сценический / ритуальный режим

## Git

Проект уже под git.
Коммиты делаются по смысловым этапам, а не после каждой мелочи.

Нормальные этапы для коммита:

- индекс обновлен
- `chapters_full.json` пересобран
- сайт заметно продвинут
- видео-конвейер расширен или стабилизирован
